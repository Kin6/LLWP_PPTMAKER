# Changelog

本项目的重要变化记录在此。当前仓库尚未建立正式版本发布规则，因此新功能先记录在 `Unreleased`，待首次标记版本时再整体归档。

## [Unreleased] - 2026-07-17

### Added

- 新增 Claude Design 风格的 HTML 交互演示模式，支持 Reveal.js 放映、ECharts 图表、动画与受控交互。
- 新增单侧栏设计工作台，支持页面切换、元素选择、拖拽缩放、属性编辑、撤销重做、评论、Tweaks、手绘标注和 AI 局部修改。
- 新增结构化 `HtmlDeckSpec`、Zod/JSON Schema 校验、白名单 Patch、sandbox iframe 运行时、IndexedDB 自动保存、单文件 HTML 导出和静态 PPTX 导出。
- 新增 DOCX 与 PDF 材料导入，保留章节、页码、表格、图片、提取方式与 OCR 置信度；扫描页按需使用浏览器端 OCR。
- 新增稳定来源块与逐页 `sourceRefs`/`sourceNotes`，让生成内容能够追溯到原始材料。
- 新增 HTML 流程、来源溯源、整页导出、图片画幅与图片提示词的自动化测试。
- 新增跨平台协作者复刻指南，覆盖安装、模拟服务、真实 API 联调、架构职责与故障排查。

### Changed

- 内容策划改为 `DeckOutline -> DeckSpec` 两轮流程，先确定严格页数和证据顺序，再扩展页面内容，降低首步输出量与等待时间。
- GPT Image 2 改为按页请求、默认跟随总页数，并在每页成功后立即保存检查点；返回图片统一整理为 1536 x 864 的 16:9 画布。
- 工作流改为逐阶段流式显示，后续步骤只在实际到达时出现，同时实时更新阶段说明与 API 调用数。
- 新建 HTML 任务时预览区保持空白，直到有效结果到达，避免显示旧稿或错误页数的默认内容。
- 拆分应用状态、工作流、首页、工作区面板与 HTML Deck 模块，缩小顶层组件职责并保持原有模式可用。
- API 凭据、Base URL 和模型改为只从服务进程的系统环境变量读取；浏览器不再显示、保存或提交敏感配置，服务端忽略客户端伪造字段。

### Fixed

- 修复 HTML 工作台点击中间页缩略图无法切换页面的问题。
- 修复交互演示第三阶段容易因模型超时、异常 JSON 或 Schema 不合格而中断的问题，增加结构化输出、修复回退与本地草稿保底。
- 修复用户请求 9 页时预览出现 7 页默认稿的问题，并严格校验 1-50 页目标数量。
- 修复流式任务轨迹提前展示未开始步骤、首屏残留旧内容和图片阶段失败后重复请求已完成页面的问题。
- 修复第三方图片网关超时、524 和临时错误时缺少重试与备用线路切换的问题。
- 修复 HTML Deck 并发自动保存时旧 revision 可能覆盖新编辑的问题。
- 修复雷达图在 HTML 运行时、属性面板和静态 PPTX 之间契约不一致的问题。
- 修复静态 PPTX 居中文本使用无效垂直对齐值，以及 `draw` 动画和动画延迟未生效的问题。
- 修正第三阶段状态说明，只报告实际执行的页数核对与 16:9 归一化。
